# Contributing

## Student contributions

Contributions should improve accuracy, clarity or usefulness.

1. Create a branch.
2. Make one focused change.
3. Test commands in an authorized lab.
4. Do not include credentials or private information.
5. Update references.
6. Submit a pull request describing the change and validation.

## Style

- Use clear headings.
- Use numbered steps for procedures.
- Use fenced code blocks with language identifiers.
- Mark placeholders with angle brackets.
- Link to official documentation.
- Include warnings before risky actions.
