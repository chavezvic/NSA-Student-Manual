# Lab Standards

## Before the lab

1. Read the objective and success criteria.
2. Confirm the authorized IP range, devices and credentials.
3. Record the initial state.
4. Create a snapshot or backup when appropriate.
5. Prepare a rollback method.
6. Verify that required installation media and licenses are available.

## During the lab

- Use descriptive hostnames.
- Keep an IP address table.
- Change one major variable at a time.
- Record commands that modify the system.
- Capture exact errors.
- Label screenshots.
- Validate every dependency before continuing.
- Do not disable security controls without documenting the reason and restoration step.

## After the lab

1. Test functionality.
2. Test security.
3. Test restart persistence.
4. Review logs.
5. Record the final configuration.
6. Remove temporary accounts, rules and files.
7. Confirm backups.
8. Write a short reflection.

## Minimum evidence

- Objective
- Scope
- Diagram
- Addressing or inventory table
- Procedure
- Commands
- Validation results
- Screenshots where useful
- Problems and resolutions
- Final state
- Rollback or recovery notes
- Reflection
